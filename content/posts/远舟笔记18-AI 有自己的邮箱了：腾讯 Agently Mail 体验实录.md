>远舟笔记·第18篇

打开 QQ 邮箱订阅号，看到一条推送：**「Agently Mail，AI 的专属邮箱来了」。**

没多想，就试了一下。 体验地址：https://agent.qq.com/

## 跑通它比我想象的简单

安装 CLI：

```bash
npm install -g @tencent-qqmail/agently-cli
```

安装 Skill——让 Hermes 能调用邮件功能的桥梁。第一次执行卡住了，换了个新终端重试，一次成功。

授权更简单，终端输出一个 URL，浏览器打开确认即可。

```bash
agently-cli auth login
```

3分钟不到，我的 AI Agent 就拥有了一个邮箱。

## 途中遇到一个问题

真正让我停下来的是两阶段确认机制。

发送邮件时，第一次调用不会直接发出，而是返回一个确认令牌。用户确认后带上令牌再执行第二次，邮件才真正发出。
![file-20260623170251379](/images/file-20260623170251379.jpg)
**读取邮件 ≈ 获取信息，发送邮件 ≈ 对外行动。**

Agent 的风险不是看，而是做。

腾讯在这里用了 Human in the Loop（人在回路中）——让 AI 能做事，但关键动作必须有人确认。

这在传统邮件客户端里是多余的。但在 AI Agent 的语境下，它是必要的安全底线。

## 腾讯的AI生态版图再添一员

Agently Mail 不是一个给人类用的邮件客户端。

它是一个 CLI 工具，这意味着任何能执行 shell 命令的 Agent 框架——Hermes、WorkBuddy、Claude Code、Kimi Work、豆包超能模式——都可以调用它。

它不挑 Agent，只挑有没有 shell 权限。

我把它放在腾讯的AI产品线里看，从腾讯元宝到ima Copilot，从混元大模型到各类AI原生工具，腾讯一直在布局AI的基础能力。Agently Mail 是这条线上的新成员——它不是锦上添花的小工具，而是给 AI Agent 修建的一条邮件高速公路。

现在路上车还很少，但路已经修好了。

更值得期待的是它未来与企业微信邮箱之间的联动。如果 Agently Mail 能接入企业微信邮箱，那就意味着企业的 AI Agent 可以直接处理商务邮件——报价、确认、跟进、归档——这些场景一旦跑通，就是真正的效率革命。
